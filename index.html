1 Requirements

Architectural

Organizations must be able to run their own instance of the system

The system must allow any organization to install the software on their own server and database, resulting in their own instance.

 

Users of this instance must not see the data of any other instance in any user interface, except for reuse of SSI-related credentials such as in the Sovrin wallet.

 

Organizations must be able to setup their instance with configuration of the system only, without the need for changing any code.

System must use the verifiable credential standard (https://www.w3.org/TR/vc-data-model/)

 

Architectural

7.1.2 Person affected status

Figure 63 : Person affected status

Statuses that a Person affected can have throughout a program. These statuses are 'calculated' when requested and not stored in a database.

Validated

The Person affected is validated and has a credential for identity and program.

Selected for validation

The Person affected is selected for validation.

Created

The Person affected created an account, but did not register for an identity and program yet.

REFACTOR: propose to rename to Created account

Included

The Person affected is included for installment and will receive payments.

Rejected

The Person affected is manually rejected from receiving installments and will not receive payments.

Registered

The Person affected registered for an identity and program.

 

7.1.3 Program phases

Figure 64 : Program phases

 

Review inclusion

Inclusion phase closed. List of included people is printed and agreed upon in the community.

 

(The Program Manager can still include and exclude Persons affected.)

Registration and validation

Program is published on the Sovrin blockchain.

Persons affected can register and can be validated, this is called Registration in the UI.

Evaluation

This is a "read-only" phase where no changes can be made anymore.

Design

Program is in the design phase. Persons affected cannot register.

Payment

One or more installments are made.

 

(The Program Manager can still include and exclude Persons affected.)

 

Inclusion

Registration and validated are closed, persons affected can be included.
